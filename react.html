What is react?

React is a view layer library written in javascript, not a framework like Backbone, Angular etc. It was developed by Facebook. You can't use React to build a fully-functional web app, as it has no server layer.

React in the wild

Why was react created?

  • Shift from MVC mentality
  • Need for a more modular component approach
  • The complexity of 2 way data binding
  • Increased speed of reactivity

Terms you have to know

  • Components
  • Props
  • State
  • JSX
  • Virtual DOM

Let's dive on in

Components

Components are self contained pieces of code that are used to build up a web app. They are the building blocks of your app. I like to think of them as lego pieces.

React components are basically just pure functions (same input produces same output). There are exceptions, to this statement, but generally this is the case.

Example

          
            const Simple = () => {
              return 
Hello World!
;
            };
            
            export default Simple;

Props

Props are the properties passed to the component from it's parent.

This is the data for the component to use to render.

Example

          
          const Props = (props) => { // or ({name})
            if (props.name) {
              return 
Hello {props.name}!
;
            }
            return 
Hello World!
;
          };

          export default Props;

          // Used like so
          <Props />
          <Props name="Randy" />

State

This is the internal state of the component, when a state object changes it causes the component to re-render.

A state object is added by using a useState hook(more on this later). A state object can be as simple as a boolean or a deeply complex object.

Example

JSX

I frankly don't even know how to really describe this one. JSX is a syntax extension of javascript. It's a bit like writing html inside of javascript, but so much more.

You can also include other components in it like any other standard html elements.

You can even embed ANY javascript expression in JSX, this turns it into an extremely strong templating language.

Virtual DOM

The virtual DOM is used for efficient re-rendering of the DOM

React aims to re-render the virtual tree only when the state changes

Uses 2 virtual trees (new and previous) to find differences and batch update real DOM

Observes data changes (setState, again more on this later) and does dirty-checking to know when to re-render component

Whenever possible, does not update entire component in real DOM - only computes a patch operation that updates part of the DOM

useState

The useState hook uses something called destructuring in javascript to give you 2 things

  1. A variable that has the state object
  2. A method to call to update the state object

Why does it return an array? Well it's a DX thing to allow for easier naming. When a function returns an object everything has a name, an array allows you to easily names things how you want.
