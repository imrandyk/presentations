Azure Functions

What are they?

“Azure Functions is a serverless solution that allows you to write less code, maintain less infrastructure, and save on costs.”

Azure Functions is a hosted server in the runtime of your choosing that you do not have to deal with from a hardware and networking perspective. With the way functions work, and the fact they are atomic there is less code needed since you don't need to wire up any routing or startup type code. Because the hardware can be shared, there is a cost reduction that comes with that.

Pretty simple, at the highest level concept

TIMER

What is it?

A function that happens periodically, based on a CRON statement

Keep in mind azure goes down to the second and is 24 hour based, other CRON systems are limited to the minute

Function Parts

CRON Statements

  1. */30 * * * * * - every 30 seconds
  2. 0 */5 * * * * - every 5 minutes
  3. 0 5 4 * * * - at 04:05 every day
  4. 0 5 0 * 8 * - at 00:05 in August
  5. 0 15 14 1 * * - at 14:15 on day 1 of every month
  6. 0 0 22 * * 1-5 - at 22:00 on Monday through Friday of every month

When to Use

  • Any time something needs to happen daily, weekly, monthly, ect
  • When the load of the work is to demanding for real time event processing

HTTP

What is it?

A function that happens when it is invoked by a HTTP call

All HTTP methods are allowed

Function Parts

When to Use

  • There is a need for an API
  • When you are needing to connect modern tools to legacy workloads
  • Provide internal data to external entities

BLOB

What is it?

A function that happens a new blob is added to a container

Think of a container as a folder in azure, limited to 50MB

Function Parts

You can even limit the file

When to Use

  • There is a file involved

QUEUE

What is it?

A function that happens am item is added to a queue

Think of a queue as a list of items all the same but not associated

Function Parts

When to Use
