What makes up a CSS rule?

A selector and declarations, which are a list of property-value pairs

You can apply the same declarations to multiple selectors.
h1 {
  font-size: 25px;
  color: red;
}
Currently there are 176 CSS properties, this is not counting vendor-prefixed properties or non-shorthand properties. More are being added by vendors everyday.

You can define your CSS in 3 ways

  • Inline - No selector is needed
  • Internally
  • Externally

Inline

Inline declarations are done on a HTML element via the style attribute. Since it is done directly on the element no selector is needed.


  Hello World

Internally

Internally you can define CSS at the document level by putting your rules in a style tag in the head of the document.

<!DOCTYPE html>
<html>
  <head>
    <style>
      h1 {
        font-size: 25px;
        color: red;
      } 
    </style>
  </head>
  <body>
    <h1>Hello World</h1>
  </body>
</html>

Externally

Externally you can define CSS in a different file and include it into the document via a link tag in the head of the document.

<!DOCTYPE html>
<html>
  <head>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <h1>Hello World</h1>
  </body>
</html>

What do you thing the output would look like now?
Lets look at how the browser handles this.
  • Make all h1s red with a font size of 25 pixels and keep any other properties.
  • Make all divs have a font size of 50 pixels and keep any other properties.
  • Make all divs that are under any element with a class of 'red', red and make the font size 10 pixels and keep any other properties.
  • Make all divs that are under any h1 with a class of 'red' purple and keep any other properties.
  • Make the element with an id of 'world', green and keep any other properties.

Specificity

The way specificity works is the browser will take a rule and give it a ranking, it applies the rules to an element from lowest rank to highest rank. This ranking can be calculated pretty quickly once you understand it.

Specificity Ranking

  • Type Selectors - Get 1 point
  • Class Selectors - Get 10 points
  • Id Selectors - Get 100 points

Let's look at our 3 rules we added to our example earlier.

So what is the !important tag?

!important

The !import tag allows you to force a property to be what you want it to be. This is not a good practice as it makes debugging your style rules extremely difficult. You can typically find a way to increase the specificity of the rule you are wanting to apply instead.

But there is always that 1 exception!

!important

If you are using a plugin or component that uses inline styles, which in itself is a bad practice, the only way to change that may be with the use of !important.
